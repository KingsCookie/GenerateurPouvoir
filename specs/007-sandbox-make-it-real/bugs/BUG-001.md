## Description

L'attente : « la création et l'édition d'une personne permet de **TOUT FAIRE** » — à la fois
l'**espèce**, le **genre**, le **nom** et le **statut** (vivant / mort), mais aussi le **cycle de vie**
(conjoints, ex, divorce, etc.), les **traits** (ADN) et le **profil de pouvoir** (sans-pouvoirs /
mutation forte / mutation normale).

Le formulaire implémenté ([SandboxView.svelte:44-91](../../../src/ui/views/SandboxView.svelte#L44-L91))
n'expose en réalité que **5 champs** : `nom`, `especeId`, `genreId`, `notes`, `vivant`. Manquent :

- **ADN / traits** (`adn.traits`) — pourtant accepté par le cœur (`PersonDraft.adn`).
- **Pouvoirs** (`pouvoirs[]`, puissance/maîtrise) et le **profil de génération** (sans-pouvoirs,
  mutation forte, mutation normale) — accepté par le cœur (`PersonDraft.pouvoirs`).
- **`raisonDeces`** quand l'individu est marqué mort — accepté par le cœur (`PersonDraft.raisonDeces`).
- **Cycle de vie / conjoints** (actuel, ex, divorce) — **ni** dans le cœur **ni** dans l'UI
  (volontairement exclu de `PersonDraft` / `PersonPatch`).

On distingue donc deux défauts :

- **(A) Dérive d'implémentation** : `adn`, `pouvoirs`, `raisonDeces` sont **exigés éditables** par la
  spec (FR-010, FR-011a) et **supportés par le cœur**, mais **absents du formulaire UI**.
- **(B) Lacune de spec** : l'édition directe du **cycle de vie / conjoints** (actuel ↔ ex ↔ divorce ↔
  célibataire) n'est **couverte par aucune exigence** et est **explicitement exclue** du modèle
  (`PersonDraft` n'inclut pas `conjoints` ; INV-S5 interdit à `editPerson` de toucher `conjoints`).

- **Lacune identifiée** : aucune FR ne régit l'édition directe du **statut de couple/conjoint**
  (établir/dissoudre un couple, marquer « ex », divorcer) dans la sandbox.

## Root Cause Analysis

- **(A)** Oubli de portée à l'implémentation UI : le contrat cœur (`PersonDraft`) prévoyait `adn` et
  `pouvoirs`, mais le formulaire `SandboxView` n'a livré qu'un sous-ensemble minimal (l'astuce « clonez
  puis éditez » dans le formulaire confirme que la saisie fine d'ADN/pouvoirs a été sciemment reportée).
  C'est une **dérive d'implémentation** : le code UI diverge de la spec déjà écrite.
- **(B)** **Lacune de spec** : l'édition directe du cycle de vie/conjoints n'a jamais été spécifiée ;
  la conception a au contraire restreint l'édition aux **attributs** (INV-S5) en réservant la parenté à
  la reproduction. La demande élargit le périmètre au-delà de l'existant — il faut amender la spec avant
  d'implémenter.
