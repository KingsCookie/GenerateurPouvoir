# Bug Report: Les filtres de la liste de population sont absents de la page sandbox

**Type**: Spec gap
**Severity**: Medium
**Feature**: 007-sandbox-make-it-real
**Reported**: 2026-06-17
**Status**: Patched
**Patched**: 2026-06-17 — Spec : FR-021 (parité de filtrage) + SC-013. Research : D12. Plan : ligne de
décision « Filtres sandbox ». Contracts : note frontière UI. Tasks : T028 (UI filtres sandbox).

## Description

La liste de population (page principale) propose une **barre de filtres** ([FilterBar.svelte](../../../src/ui/components/FilterBar.svelte))
adossée au moteur pur `filterPopulation` (Feature 4/5) : recherche par nom, espèce/genre, génération,
modes d'affichage des traits, etc. ([ListeView.svelte:3-16](../../../src/ui/views/ListeView.svelte#L3-L16)).

La **page sandbox** ([SandboxView.svelte](../../../src/ui/views/SandboxView.svelte)) réutilise bien
`buildListRow` pour le rendu, mais **n'inclut ni `FilterBar` ni `filterPopulation`** : la liste sandbox
n'est pas filtrable. L'attendu : **les mêmes filtres que la liste de population doivent être présents
dans la page sandbox**.

## Artifact Traceability

### spec.md

- **User story affectée** : transverse à **US1** (bac à sable isolé — affichage de la population copiée)
  et **US3** (navigation temporelle — affichage de l'état reconstruit à l'année).

## Root Cause Analysis

**Lacune de spec** : la spec de la Feature 7 décrit l'écran sandbox comme une liste d'individus avec les
actions (créer/cloner/éditer/supprimer, reproduction, année) mais n'a **jamais exigé** la réutilisation
des filtres de la Feature 4/5. L'implémentation a suivi la spec à la lettre (liste non filtrée). La
parité de filtrage est une attente légitime non capturée — d'où l'amendement nécessaire avant correctif.
