# Bug Report: Arbre — finitions visuelles (liens couple↔⚭, liens en équerre, conjoints grisés, décédés colorés, légende)

**Type**: Spec gap (exigences de rendu visuel)
**Severity**: Low
**Feature**: 004-genealogie-exploration
**Reported**: 2026-06-10
**Status**: Patched
**Patched**: 2026-06-10

## Description

Cinq finitions de lisibilité du rendu de l'arbre (fiche **et** page dédiée), après la refonte SVG
(BUG-004).

1. **Traits entre les membres d'un couple et le symbole ⚭.** Actuellement le ⚭ « flotte » entre les
   deux cases sans trait. Il faut **relier chaque membre du couple au symbole ⚭** par un petit
   segment (case ↔ ⚭ ↔ case).
2. **Liens couple → enfants en équerre (3 segments, 2 angles droits)** au lieu de lignes droites :
   - **Segment 1 (vertical)** : du **symbole ⚭** jusqu'à **un peu au-dessus du niveau des enfants** ;
   - **Segment 2 (horizontal)** : de ce point jusqu'**au-dessus de l'enfant** à relier ;
   - **Segment 3 (vertical)** : de **un peu au-dessus de l'enfant** jusqu'à **l'enfant**.
   (Routage orthogonal classique avec une **barre horizontale** commune par fratrie.)
3. **Conjoints des descendants grisés.** Les **conjoints** (pièces rapportées) des descendants de la
   personne ciblée — et le conjoint de la racine — DEVRAIENT être **grisés** pour les distinguer des
   membres **du sang** de la famille.
4. **Personnes décédées d'une autre couleur.** Les individus **décédés** DEVRAIENT avoir une
   **couleur distincte**, **sans** impacter le reste de l'arbre (juste leur case).
5. **Légende.** Ajouter une **légende** sur **les deux pages** (fiche **et** page dédiée d'exploration)
   expliquant les symboles/couleurs (⚭ union, ⚮ / pointillés = ex, grisé = conjoint « pièce
   rapportée », couleur racine = personne observée, couleur = décédé).

## Décisions validées (utilisateur, 2026-06-10)

- **Addition `vivant` au cœur** : ✅ retenue — `vivant: boolean` ajouté à `TreeNodeLite` (pur,
  déterministe, lecture seule).
- **Styles cumulables** (un même individu peut combiner plusieurs états) — chaque état utilise une
  **dimension CSS distincte** pour rester combinable :
  - **ex** ⇒ **bordure en pointillés** (`border-style: dashed`) — *(rappel : inchangé)* ;
  - **conjoint / pièce rapportée** ⇒ **fond grisé** (`background`) ;
  - **décédé** ⇒ **couleur distincte** via **couleur de bordure** + **marqueur « † »** (n'affecte
    que sa case) ;
  - **racine** ⇒ **accentuation** (fond + bordure accent).
  - Exemple combiné : un **ex décédé pièce rapportée** = **fond grisé** + **bordure pointillés** de
    **couleur « décédé »** + « † ». Aucune dimension n'écrase les autres.

## Root Cause Analysis

Lacunes de **spécification visuelle** : la refonte SVG (BUG-004) a fiabilisé les liens mais n'avait
pas prévu le **routage en équerre**, les **segments membre↔⚭**, ni les codes couleur **sang / pièce
rapportée / décédé**, ni la **légende**. Pour les décédés, c'est aussi un **manque de donnée** au
niveau du nœud (`vivant` absent du `TreeNode`) — d'où une petite extension du cœur. Le reste est
purement présentation (UI), sans impact sur le déterminisme ni la lecture seule.
