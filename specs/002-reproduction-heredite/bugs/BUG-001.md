# Bug Report: Écran des paramètres — absence de regroupement thématique et d'explications

**Type**: Spec gap (exigence UX manquante)
**Severity**: Medium (utilisabilité ; aucune incidence sur le moteur ni le déterminisme)
**Feature**: 002-reproduction-heredite
**Reported**: 2026-06-09
**Status**: Patched
**Patched**: 2026-06-09

## Description

L'écran des paramètres présente tous les paramètres du moteur **à plat**, sous un unique titre
« Moteur génétique », sans regroupement par finalité et **sans explication** de ce que fait chaque
paramètre (seul `A` porte une infobulle). C'est peu lisible pour un outil d'expérimentation.

Deux corrections demandées :

1. **Regrouper** les paramètres en **sections thématiques** correspondant à leur rôle, à l'image de
   l'organisation de la source de vérité (§9) :
   - **Génération de pouvoir** (§9.1) : seed, chance de pouvoir du batch, résilience initiale,
     constante de duplication `D`, constante de génération `K`.
   - **Hérédité & naissance** (§9.2) : résilience maximale, bonus (points), malus (points),
     seuil de disparition, taux de mutation forte, taux de naissance sans pouvoir,
     taux de mutation faible (gain / perte), option « malus génome », `B`, `C`, `A` (lecture seule).
   - **Population** (§9.3) : nombre d'individus, année de naissance.
2. **Décrire** chaque paramètre par un court texte d'aide **au-dessus** (ou sous) le champ de saisie —
   pas seulement pour `A`. La description doit dire **à quoi sert** le paramètre (effet concret), pas
   répéter son nom.

## Root Cause Analysis

Oubli de spécification UX : FR-027/FR-030 ont cadré **le contenu** (quels paramètres, éditabilité,
`A` calculée) mais pas **la forme** de l'écran. T021 a donc été implémentée littéralement (édition à
plat). La source de vérité §9 fournit pourtant un regroupement naturel qui aurait dû être reflété.

## Recommended Fix

1. Lancer `/speckit-bugfix-patch` pour :
   - ajouter à **spec.md** une exigence (p. ex. **FR-031**) : l'écran des paramètres DOIT **regrouper**
     les paramètres en sections thématiques (génération de pouvoir / hérédité & naissance / population,
     cf. §9.1/§9.2/§9.3) et afficher pour **chaque** paramètre une **brève description fonctionnelle** ;
   - rouvrir **T021** (ou ajouter une tâche dédiée) dans **tasks.md** pour le regroupement + les aides.
2. Lancer `/speckit-bugfix-verify` pour confirmer la cohérence des artefacts.
3. Reprendre `/speckit-implement` pour appliquer le correctif dans `ParametresView.svelte`.
